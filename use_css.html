Introduction

Cascading Style Sheets (CSS) is a style sheet language used for describing the presentation of a document written in a markup language such as HTML. CSS is designed to enable the separation of presentation and content, including layout, colors, and fonts. This separation can improve content accessibility; provide more flexibility and control in the specification of presentation characteristics; enable multiple web pages to share formatting by specifying the relevant CSS in a separate .css file, which reduces complexity and repetition in the structural content; and enable the .css file to be cached to improve the page load speed between the pages that share the file and its formatting.
Separation of formatting and content also makes it feasible to present the same markup page in different styles for different rendering methods, such as on-screen, in print, by voice (via speech-based browser or screen reader), and on Braille-based tactile devices. CSS also has rules for alternate formatting if the content is accessed on a mobile device.
The name cascading comes from the specified priority scheme to determine which style rule applies if more than one rule matches a particular element. This cascading priority scheme is predictable. The CSS specifications are maintained by the World Wide Web Consortium (W3C). The W3C operates a free CSS validation service for CSS documents.


How is it used?

CSS is applied to a HTML document in three ways: External style sheet - Linking an external CSS document with the html file; Embedded style sheet - Writing the CSS within the <style> tag inside the <head> tag; Inline style - Written explicitly inside the tag using the style attribute.
External style sheet is more commonly used in the real world as it allows the design to be easily changed without any risk of change in the HTML document. It allows the same design to be used over and over by just linking one single file to all the documents.
Inline style preceds over embedded style sheet which preceds over the external style sheet. In case of the same style sheet, the property that comes at the last preceds over all other properties.


Selectors

To apply the CSS, you need to select an element over which you want to apply the CSS. And for that, we have multiple ways to select an element.

Universal selector
* is called the universal selector. It selects all the tags in the html document and apply the CSS on all of them.
*{property:value;}
Type selector
Tag name is used select that particular tags. It selects all the tags that have the same name.
tag_name{property:value;}
ID selector
The id specified to a tag is used to select it. Its benefit is that id is unique and hence no two elements will be slected by mistake.
#id{property:value;}
Class selector
The class specified to a tag is used to select it. Its benefit is that class is not unique and hence it is used to select multiple tags at once.
.class{property:value;}
Relational selector
Descendant selector
It selects all the child elements nested within the parent element no matter how nested it is.
parent_name descendant_name{property:value;}
Child selector
It selects the child elements nested just within the parent element.
parent_name > child_name{property:value;}
Adjacent selector
It selects the first sibling of an element.
tag_name + sibling_name{property:value;}
General Sibling Selector
It selects all the siblings of an element.
parent_name ~ sibling_name{property:value;}

Conflicting Rules

Many a times, a developer may use by mistake or intentionally, conflicting rules over the same element for example specifying the color of p tag as green somewhere and red somewhere else. Then, which color will be applied? What rules will be followed?
Depending on the CSS being used external or internal it can easily be judged.
Here, is the preference order depending on the how is CSS being applied, externally or internally:

  • !important rules in user style sheet
  • !important rules in web page
  • Normal rules in web page
  • Normal rules in user style sheet
  • Default rules in web browser

Now, what if conflicting rules appear in the same CSS sheet. Here is the precedence order:

  • More specificity (tag_name.class over .class)
  • id >= class >= attribute >= pseudo-class > pseudo-element > element > universal
